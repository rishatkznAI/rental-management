Create a high-fidelity, editable Figma design for a B2B web admin system to manage an aerial work platform / lift rental business. Language: Russian. Audience: internal staff (rental managers, service team, accountant, admin). Platform: Web (desktop first), responsive down to 1366×768.

Brand & style
- Clean, professional, data-dense, industrial feel (not playful).
- Light theme default + optional dark mode styles.
- Color direction: deep navy + neutral grays; status colors for tags (available / rented / reserved / in service / inactive) in green/yellow/red/gray.
- 8px spacing grid, clear typography, strong table usability.
- Reusable components: buttons, inputs, dropdowns, date pickers, tables, chips/tags, modals, toast notifications, empty states, skeleton loaders.

Information architecture (left sidebar)
- Дашборд
- Техника
- Аренды
- Сервис
- Клиенты
- Документы
- Платежи
- Отчёты
- Настройки

Core data model (use in UI labels/fields)
Equipment unit (Техника):
- ID/Инв.№, Модель, Тип (ножничный/коленчатый/телескопический), Привод (дизель/электро),
- Серийный № (SN), Год, Высота подъёма, Локация (город/склад/объект),
- Статус (Свободен/В аренде/Бронь/В сервисе/Списан),
- Следующее ТО дата, Примечания, Фото, Документы.

Rental order (Аренда):
- Клиент (юрлицо/ИП), Контакт, Даты (старт/план возврата/факт),
- Техника (1+ единиц), Тариф (сутки/месяц), Цена, Скидка,
- Доставка (туда/обратно, адрес, окно времени), Ответственный менеджер,
- Статус: Новый → Подтверждён → Доставка → Активен → Возврат запланирован → Закрыт,
- Документы: Договор/Акт/Счёт, Комментарии, Риски/штрафы.

Service ticket (Сервис):
- Причина/описание, Приоритет, SLA, Назначен (механик),
- Работы (журнал), Запчасти, Время, Фото/вложения,
- Статус: Новый → В работе → Ожидание запчастей → Готово → Закрыто.

Client (Клиенты):
- Компания, ИНН, контакты, договорные условия, отсрочка, лимит, история аренды, задолженность.

Required screens (design all)
1) Login (Вход)
- Simple, branded, with “Войти” button and error state.

2) Dashboard (Дашборд)
- KPI cards: Утилизация парка %, Активные аренды, Просроченные возвраты, Техника в сервисе, Выручка за день/неделю.
- Alerts panel: Просрочки оплат, Возвраты на 3 дня, Критические заявки сервиса.
- Quick actions: “+ Новая аренда”, “+ Заявка в сервис”, “+ Клиент”.

3) Equipment list (Техника → Список)
- Data table with columns: Инв.№/Модель, Тип, Привод, Статус (tag), Локация, След. ТО, Текущий клиент, Дата возврата.
- Advanced filters: Статус, Тип, Привод, Локация, Дата ТО, Поиск (SN/модель/клиент).
- Bulk actions: “Отправить в сервис”, “Сменить локацию”, “Экспорт”.
- Row quick actions (…): Открыть, Сдать в аренду, Вернуть, Создать заявку.

4) Equipment detail (Карточка техники)
- Header with photo thumbnail, status tag, key specs, actions: “Сдать”, “Вернуть”, “В сервис”.
- Tabs:
  - Обзор (ключевые поля)
  - История аренды (table)
  - История сервиса (table + link to ticket)
  - Документы (upload/download)
  - Фото (gallery)
- Maintenance widget: “ТО через X дней”, schedule button.

5) Rentals (Аренды)
- Kanban board by status + list view toggle.
- Card shows: клиент, техника, даты, цена, доставка, риск (if overdue).
- “New rental” modal/wizard:
  Step 1 Клиент, Step 2 Техника (availability search), Step 3 Даты/тариф/цена, Step 4 Доставка, Step 5 Документы/подтверждение.
- Rental detail page with timeline and documents section.

6) Service (Сервис)
- Ticket list with priority/SLA badges, filters, search.
- Ticket detail:
  - Timeline of updates
  - Work logs (add entry)
  - Parts (add part, qty, cost)
  - Attachments/photos
  - Buttons: “В работу”, “Ожидание запчастей”, “Закрыть”.

7) Clients (Клиенты)
- Client list (table) with segments, debt tag, last rental date.
- Client detail:
  - Overview (contacts, terms)
  - Rentals history
  - Documents
  - Payments / debt summary

8) Documents (Документы)
- List: Договоры, Акты, Счета; status (Черновик/Подписан/Отправлен), download.
- Document preview panel.

9) Payments (Платежи)
- Table: счет/клиент/сумма/дата/статус, overdue highlights.
- Reconciliation notes.

10) Reports (Отчёты)
- Utilization by month, revenue by client, downtime reasons (simple charts + tables).

11) Settings (Настройки)
- Users & roles
- Reference data: типы техники, локации, статусы, причины простоя
- Notification rules (email/telegram concept)

UX requirements
- Fast data entry, minimal clicks, inline edit where safe.
- Clear empty states and error states.
- Consistent toasts for success/error.
- Provide a small UI kit page: components + styles + icons usage.

Deliverables
- High-fidelity mockups for all screens listed
- Components and styles organized in a “UI Kit” page
- Russian copy for all labels, buttons, and table headers
