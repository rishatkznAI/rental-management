Сделай редизайн вкладки “Аренды” в веб-приложении так, чтобы она стала планировщиком в виде диаграммы Ганта (gantt). Никакого канбана. Это главный рабочий экран.

Цель: быстро видеть занятость техники по датам, управлять арендами, фиксировать возврат и простои. Интерфейс должен быть компактным (под 1366×768), но читабельным.

1) Верхняя панель управления (sticky)
Слева направо:
- Заголовок: “Планировщик аренды”
- Переключатель масштаба: День / Неделя / Месяц
- Кнопки навигации по времени: Сегодня, ←, →
- Primary кнопка: + Новая аренда
- Secondary кнопка: Возврат техники (открывает модал/мастер возврата)
- Secondary кнопка: Отметить простой (открывает режим выделения периода простоя на выбранной технике)

2) Панель фильтрации (вторая строка, sticky)
Сделай фильтры в 1–2 строки, компактные, с поиском:
- Модель техники (dropdown + поиск)
- Менеджер (dropdown)
- Клиент (dropdown + поиск)
- Подписан УПД: Да / Нет (toggle или dropdown)
- Оплата: Оплачено / Не оплачено / Частично (segmented control)
- Статус аренды (опционально): Создана / В аренде / Возвращена / Закрыта
- Кнопки: Применить, Сброс
- Индикатор: “Показано X из Y единиц техники / аренд”

3) Основная зона — Gantt (ядро экрана)
Сделай структуру как в классическом планировщике:
- Левая фиксированная колонка (sticky) — таблица по технике
- Правая часть — временная сетка с полосами (timeline)

3.1 Левая колонка (строки техники)
Каждая строка = 1 единица техники:
- INV-xxx + модель (жирно)
- Доп. строка мелким: тип (ножничный/коленчатый/телескопический/мачтовый), город/филиал (если есть), текущая локация
- Мини-метки: статус техники (Свободна/В аренде/В сервисе/Простой)
- Иконки быстрых действий (справа в строке):
  - + (создать аренду на эту технику)
  - ↩ (возврат — доступно если техника в аренде)
  - ⏸ (отметить простой — доступно если свободна или по разрешению)

3.2 Timeline (временная часть)
- Сетка дней/недель/месяцев в зависимости от выбранного масштаба
- Полосы по времени:
  A) Полоса аренды: сплошная
     - внутри текст: Клиент (коротко) + менеджер (инициалы) + признак оплаты (иконка)
     - маленькие бейджи/иконки на полосе:
       * УПД подписан (галочка)
       * Оплата: оплачено/не оплачено (иконка)
  B) Полоса простоя: отдельный стиль (штрих/полупрозрачная заливка)
     - подпись: “Простой” + причина (опционально)
  C) Полоса сервиса (если есть): отдельный стиль (например, диагональная штриховка) — можно как future-ready, но покажи хотя бы визуально.

- Конфликты:
  Если по одной технике две полосы пересекаются — подсветить красным, поставить “!” и показать тултип “Конфликт дат (ссылка/переход к аренде)”.

4) Взаимодействия (обязательно показать на макете)
- Клик по полосе аренды → открывается правый drawer “Карточка аренды”
- Drag & drop перенос/изменение дат:
  - Разрешено только если нет конфликтов
  - При конфликте показывать красный превью-слой и запрещать сохранение
- Создание аренды:
  - через + Новая аренда (общая кнопка)
  - через + в строке техники (с предзаполненной техникой)
  - через выделение диапазона по timeline (если реализуешь)

5) Правый drawer (обязателен, 35–40% ширины)
Открывается при клике по аренде.
Секции:
1) Заголовок: Клиент + № аренды, техника (INV-xxx), статус аренды
2) Даты: старт / окончание (только даты), кнопка “Перенести” (если права позволяют)
3) Оплата:
   - статус: оплачено/не оплачено/частично
   - сумма
   - дата ожидаемой оплаты
4) Документы:
   - УПД: подписан/не подписан (toggle) + дата
   - (если нужно) счёт: создан/не создан
5) Ответственный менеджер
6) Комментарии / история изменений
7) CTA кнопки внизу drawer:
   - Сменить статус (контекстно)
   - Возврат техники (быстрая кнопка, если “В аренде”)
   - Закрыть аренду (если доступно)
   - Создать простой (если нужно)

6) Модальные окна/режимы
6.1 “Возврат техники”
- Выбор аренды (поиск) или из открытой карточки
- Факт возврата: дата возврата (по умолчанию сегодня)
- Результат: техника стала свободна или отправлена в сервис/простой (выбор)

6.2 “Отметить простой”
- Пользователь выбирает технику и период простоя (только даты)
- Причина (опционально): нет спроса / ожидание ремонта / другое
- Сохраняет → на gantt появляется полоса простоя

7) Состояния и UX
- Sticky header (верхняя панель + фильтры + шкала времени)
- Loading skeleton для списка техники и timeline
- Empty state: “Нет данных по фильтрам” + Сброс
- Tooltip на иконках УПД/оплаты/конфликта
- Плотный режим строк (compact)

8) Итог
Сделай 2–3 макета:
- Неделя (основной)
- Месяц (обзорный)
- День (детальный)
И один макет с открытым drawer и показанным конфликтом/неоплатой/неподписанным УПД.
Стиль: строгий B2B SaaS, практичный, минимум декоративности, максимальная информативность.
