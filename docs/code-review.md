# Code Review Checklist

Use this checklist before committing changes in Skytech Rental Management.

## Business Links

- Проверена ли связь сущностей по ID, а не по названию клиента.
- Не появились ли новые связи долгов, аренд, платежей, документов или сервиса через `client`, `clientName` или `company`.
- Сохраняются ли `clientId`, `rentalId`, `equipmentId` при создании и редактировании записей.
- Учитывается ли, что старые JSON-записи могут не иметь новых полей.

## Roles And Access

- Не сломана ли ролевая модель.
- Проверки доступа находятся на backend, а frontend используется только для UX.
- Не появился ли доступ к чужим данным.
- Не раскрываются ли финансовые данные перевозчику, механику или другой неподходящей роли.
- Перевозчик видит только свои доставки по `carrierId`.
- Bot-only пользователь перевозчика не может войти во frontend.

## Business Flows

- Не сломана ли логика MAX-бота.
- Смена роли в MAX-боте сбрасывает сценарий/сессию.
- Завершенные и отмененные доставки не отображаются как активные задачи перевозчика.
- Не сломаны ли статусы техники при аренде, сервисе, отгрузке и возврате.
- Не сломаны ли платежи, документы и дебиторка.
- Финансовые отчеты группируют долг по `clientId`, а не по названию.

## Tests And Verification

- Есть ли тест на исправленный баг или хотя бы диагностический сценарий проверки.
- Проходит ли `npm test`.
- Проходит ли `node --test tests/*.test.js`.
- Проходит ли `npm run build`.
- Для frontend-сценариев рассмотрен ли `npm run test:e2e`.

## Change Hygiene

- Нет ли больших несвязанных изменений.
- Нет ли форматирования всего проекта без необходимости.
- Нет ли изменений схемы данных без явной причины.
- Нет ли массовых изменений данных без backup/export плана.
- Не добавлены ли шумные комментарии к очевидному коду.
