# Production Readiness

Дата оценки: 2026-05-02.

Оценка основана на текущем коде проекта. Статусы:

- `OK` — базовая production-готовность есть, риски локальные.
- `Partial` — основная функциональность реализована, но есть незакрытые сценарии или разрыв между frontend/backend.
- `Risk` — есть существенный production-риск для данных, доступа, эксплуатации или регрессий.
- `Missing` — важный слой или сценарий отсутствует.

Текущий общий сигнал по проверкам: `npm run build` проходит, `node --test tests/finance-core.test.js` проходит. Полный `npm test` сейчас падает на 2 тестах, не относящихся напрямую к последним изменениям: `tests/frontend-numeric-source.test.js` ожидает старый frontend-паттерн в документах, `tests/rental-change-requests.test.js` ожидает дату `2026-04-30`, фактически получает `2026-04-25`.

## Приоритеты Перед Следующей Разработкой

1. Сделать полный `npm test` зелёным: документы source-guard test и rental-change date regression.
2. Зафиксировать единую модель rental/classic/gantt и статусы техники вокруг аренды/возврата/сервиса.
3. Убрать drift между frontend/backend finance и permissions через общий источник или parity tests.
4. Добавить E2E по платежам/финансам/import-export/доставке перевозчика.
5. Документировать Railway deployment с persistent SQLite volume и обязательным backup/restore процессом.
